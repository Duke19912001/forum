[Data: Thursday, 29 Sep 2016, 08:22]
So after about 3 days of searching about
Crashes, freezes, sound distortion, graphical corruption, artifacts, flickering, crashing when loading battle, freezing in battle, crashing after battle, LumaCEG (1) has crashed
atidxx32.dll xaudio2_6.dll
appcrash BEX Rome2.exe
I figured out what was wrong: set CPU affinity to only 2 cores.

I made 2 batch files to do this automatically:
SSE

Code: Select all | Line number On/Off | Expand/Contract
cmd.exe /c start "Total War: Rome II" /affinity 3 "[SSE's FILEPATH]\SSELauncher.exe" -appid 214950
Replace [SSE's FILEPATH] with where you have installed SSE.
LumaCEG

Code: Select all | Line number On/Off | Expand/Contract
cmd.exe /c start "Total War: Rome II" /affinity 3 "[ROME 2's FILEPATH]\Total War - Rome II\Rome2.exe"
Replace [ROME 2's FILEPATH] with where you have installed the game.

These make the game run on only the first 2 cores, and have fixed every problem I had, except the game not responding when I try to exit, but that's fine because I can just end the task. I fixed this about a month ago, but I hope this helps anyone else having that has the same issue.

I don't know much about the Mod Manager but you could always do it manually by using the script here:
Code: Select all | Line number On/Off | Expand/Contract
%appdata%\The Creative Assembly\Rome2\scripts\user.script.txt


This is the format:

Code: Select all | Line number On/Off | Expand/Contract

So, an example would be:

Code: Select all | Line number On/Off | Expand/Contract
